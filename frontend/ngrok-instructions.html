Setting up ngrok for QR Code Mobile Access

This guide will help you set up ngrok to create a public URL for your local development server, allowing mobile devices to access your booking confirmation pages via QR codes.

Note: ngrok provides temporary public URLs that forward to your local development server. These URLs change each time you restart ngrok unless you have a paid account.

Step 1: Download ngrok

First, download ngrok from the official website:

https://ngrok.com/download

Choose the version that matches your operating system (Windows, macOS, Linux).

Step 2: Extract the Downloaded File

Extract the downloaded zip file to a location of your choice. You'll get a single executable file named ngrok or ngrok.exe (on Windows).

Step 3: Start Your Next.js Development Server

Make sure your Next.js development server is running on port 3000:

cd frontend
npm run dev

Step 4: Start ngrok

Open a new terminal/command prompt window, navigate to the folder where you extracted ngrok, and run:

ngrok http 3000

You should see output similar to this:

Session Status                online
Account                       (No account)
Version                       3.3.1
Region                        United States (us)
Latency                       51ms
Web Interface                 http://127.0.0.1:4040
Forwarding                    https://abcd-123-45-678-90.ngrok-free.app -> http://localhost:3000

Step 5: Copy the Forwarding URL

Copy the HTTPS forwarding URL from the ngrok output (e.g., https://abcd-123-45-678-90.ngrok-free.app).

Step 6: Use the URL in the QR Code Modal

When generating a QR code in the application:

  1. Click on "Generate QR Code" for any booking
  2. In the QR code modal, paste the ngrok URL into the "Public URL" field
  3. The QR code will now be generated with the public ngrok URL
  4. This QR code can be scanned from any mobile device to access the booking confirmation

Important Notes:

  • The free version of ngrok provides randomly generated URLs that change each time you restart ngrok
  • You need to update the URL in the QR code modal each time you restart ngrok
  • Keep the ngrok terminal window open while you're using the public URL
  • If you close the ngrok terminal or your computer goes to sleep, the public URL will stop working

Troubleshooting

ngrok Command Not Found

If you get a "command not found" error, you need to run ngrok from the directory where the executable is located, or add it to your system's PATH.

Port Already in Use

If port 3000 is already in use by another application, you can change the port in your Next.js configuration and update the ngrok command accordingly.

QR Code Not Working

Make sure:

  • The ngrok tunnel is still running
  • Your Next.js development server is running
  • You've entered the correct ngrok URL in the QR code modal
